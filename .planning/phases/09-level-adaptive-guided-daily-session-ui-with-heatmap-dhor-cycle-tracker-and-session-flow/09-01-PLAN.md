Add the following new persisted fields to MadinahHifzState interface:
- `lastSessionDate: string | null` — ISO date of last completed session (for missed day detection)
- `completedSessionDates: string[]` — rolling 90-day window of ISO dates (for activity heatmap/streak)
- `previousLevel: StudentLevel | null` — for detecting level transitions
- `sessionHistory: SessionRecord[]` — last 90 session records

Add a new `SessionRecord` interface (define above MadinahHifzState):
```typescript
interface SessionRecord {
  date: string           // ISO date
  level: StudentLevel
  sabaqPages: number
  sabqiPages: number
  dhorPages: number
  ratings: Record<number, number>  // juz -> quality 1-5
  totalMinutes: number
}
```

Add a new `completeSession` action:
```typescript
completeSession: (ratings: Record<number, number>, totalMinutes: number) => void
```
Implementation:
1. Get current state
2. Compute today's ISO date
3. Update juzQualityScores using exponential moving average: `newScore = oldScore * 0.3 + newRating * 0.7` for each rated juz
4. Build a SessionRecord from todaySession data (sabaq pages, sabqi pages, dhor pages) + ratings + totalMinutes
5. Append to sessionHistory, cap at 90 entries (keep most recent)
6. Append today to completedSessionDates, cap at 90 entries
7. Increment dhorDayNumber by 1
8. Set lastSessionDate to today
9. Check level transition: if previousLevel differs from current studentLevel after regeneration, keep the new level stored as previousLevel
10. Call set() with all updated fields
11. Call generateToday() via setTimeout to regenerate tomorrow's session

Add a `detectMissedDays` getter (not an action, a computed helper):
```typescript
getMissedDays: () => number
```
Implementation: Compare lastSessionDate with today. If null return 0. Calculate day difference minus 1 (yesterday is expected). Return max(0, diff - 1).

Add a `levelTransitionDetected` derived field:
```typescript
levelTransitionDetected: boolean  // true when previousLevel !== null && previousLevel !== studentLevel
```
Set this in generateToday() after computing the new level — compare with previousLevel stored in state.

Update `partialize` to include all new persisted fields: lastSessionDate, completedSessionDates, previousLevel, sessionHistory.

Set defaults for new fields: lastSessionDate: null, completedSessionDates: [], previousLevel: null, sessionHistory: [], levelTransitionDetected: false.

Update generateToday() to set previousLevel on first run (if previousLevel is null and studentLevel is computed, set previousLevel = studentLevel). On subsequent runs, compare: if previousLevel !== newLevel, set levelTransitionDetected = true and update previousLevel to newLevel.

Add `dismissLevelTransition` action that sets levelTransitionDetected to false.
