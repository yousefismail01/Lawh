**SessionTierCard.tsx:**
A card representing one tier (sabaq/sabqi/dhor) in the session flow. Props:
```typescript
interface SessionTierCardProps {
  tier: 'sabaq' | 'sabqi' | 'dhor'
  assignment: string           // e.g., "Juz 5, pages 82-86"
  pages: number
  status: 'upcoming' | 'active' | 'completed'
  rating?: number              // 1-5 if completed
  isPaused?: boolean           // for sabaq pause state
  pauseReason?: string         // from getSabaqAllowance().reason
  onMarkComplete: () => void
  onRate?: (score: number) => void  // for sabqi/dhor
}
```

Visual states:
- `upcoming`: dimmed (opacity 0.5), no interaction
- `active`: full opacity, highlighted border (accent color), shows "Mark Complete" button for sabaq or QualityRating for sabqi/dhor
- `completed`: green checkmark overlay, shows rated score

Use tier colors from TodaySession (sabaq=green, sabqi=blue, dhor=amber). Use Ionicons (book-outline, reload-outline, library-outline). Show page count pill. When isPaused=true for sabaq, show a warm card with text: "{pauseReason}" and a small hadith snippet: "Commit yourselves to the Quran, for it escapes faster than a camel from its tying rope."

For sabqi and dhor, the "complete" action is rating via QualityRating. When the user taps a rating, that IS the completion signal — no separate "Mark Complete" button needed. For sabaq, show a single "I've completed my sabaq" button (no quality rating — sabaq progress is page-based, not quality-based).

**session.tsx:**
A new screen at `app/(main)/session.tsx`. This is a state-machine-driven guided session.

State machine (use useState, not a library):
```typescript
type SessionPhase = 'sabaq' | 'sabqi' | 'dhor' | 'summary'
// sabqi and dhor may have multiple assignments — track index
```

On mount:
1. Read todaySession and studentLevel from madinahHifzStore
2. If todaySession is null, show error and go back
3. Record startTime via useRef
4. Determine starting phase: if sabaq exists, start at 'sabaq'. If no sabaq (paused or level 5), start at 'sabqi'. If no sabqi, start at 'dhor'.

Layout:
- Header: "Today's Session" + back button (with exit confirmation if in progress) + progress indicator showing current phase
- Phase progress: horizontal step indicator (3 dots/segments for sabaq/sabqi/dhor) with current highlighted
- Main area: ScrollView with SessionTierCard for each assignment in the current phase
- For sabqi/dhor phases with multiple entries: show ALL entries for that tier, each with its own QualityRating. User rates each one. When all entries in a tier are rated, auto-advance to next tier.

State tracking:
```typescript
const [phase, setPhase] = useState<SessionPhase>('sabaq')
const [sabaqDone, setSabaqDone] = useState(false)
const [sabqiRatings, setSabqiRatings] = useState<Record<number, number>>({})  // juz -> quality
const [dhorRatings, setDhorRatings] = useState<Record<number, number>>({})
```

Phase transitions:
- sabaq complete -> if sabqi entries exist, go to 'sabqi', else if dhor entries exist go to 'dhor', else 'summary'
- all sabqi rated -> if dhor entries exist, go to 'dhor', else 'summary'
- all dhor rated -> 'summary'

When entering 'summary':
1. Compute totalMinutes from startTime
2. Merge sabqiRatings and dhorRatings into one Record<number, number>
3. Call `useMadinahHifzStore.getState().completeSession(allRatings, totalMinutes)`
4. Show MadinahSessionSummary modal
5. On modal close, check levelTransitionDetected — if true it will show in the summary (passed as prop)
6. Navigate back via router.back()

Back button behavior: if any tier is completed, show Alert confirming exit ("Progress will be saved"). If nothing done, just go back.

Use useSafeAreaInsets for top padding. Use useResolvedTheme for dark mode. Use Haptics.impactAsync(Medium) on tier completion.
