3. Create `lawh-mobile/components/mushaf/MushafScreen.tsx`:
   - This is the main mushaf experience component
   - Reads `navigationMode`, `lastReadPage`, `_hasHydrated` from `useSettingsStore`
   - **Wait for hydration:** If `!_hasHydrated`, show a loading indicator (avoids flash-of-page-1 pitfall)

   - **Horizontal mode (default):**
     - Use `PagerView` from `react-native-pager-view`
     - `layoutDirection="rtl"` for RTL book-like swiping
     - `initialPage={lastReadPage - 1}` (0-indexed)
     - `offscreenPageLimit={1}` (render current + 1 adjacent)
     - Render 604 `<MushafPage pageNumber={index + 1} />` as children
     - IMPORTANT: Do NOT render all 604 as direct children. Use a lazy approach:
       - Track `currentPage` state
       - Only render pages within range `[currentPage - 2, currentPage + 2]` as actual MushafPage components
       - Render empty placeholder Views for all other pages (PagerView needs all children but they can be empty)
       - This keeps memory usage low while PagerView handles native page scrolling
     - `onPageSelected` callback: update `currentPage` state and call `setLastReadPage(pageIndex + 1)`
     - Pass `onAyahLongPress` to each MushafPage: `<MushafPage pageNumber={...} onAyahLongPress={handleAyahLongPress} />`

   - **Vertical mode:**
     - Use `FlatList` with `data={Array.from({length: 604}, (_, i) => i + 1)}`
     - `renderItem` renders `<MushafPage pageNumber={item} onAyahLongPress={handleAyahLongPress} />`
     - `getItemLayout` for fixed page heights (enables instant scrollToIndex)
     - `initialScrollIndex={lastReadPage - 1}`
     - `windowSize={3}` for performance
     - `onViewableItemsChanged` to track current page and update `lastReadPage`

   - **Top bar:** Minimal overlay with:
     - Surah list icon (opens SurahListModal)
     - Semi-transparent background
     - Auto-hide behavior (show on tap, hide after 3s)

   - **Bottom bar:** `<PageNavigator>` with current page, slider, surah list button

   - **Jump to page:** When PageNavigator or SurahListModal triggers a page change:
     - Horizontal: `pagerViewRef.current?.setPage(page - 1)`
     - Vertical: `flatListRef.current?.scrollToIndex({ index: page - 1, animated: false })`

   ```typescript
   async getSurahStartPage(surahId: number, riwayah: Riwayah = DEFAULT_RIWAYAH): Promise<number> {
     const result = await db.select({ page: ayahs.page }).from(ayahs)
       .where(and(eq(ayahs.surahId, surahId), eq(ayahs.riwayah, riwayah)))
       .orderBy(ayahs.page)
       .limit(1)
     return result[0]?.page ?? 1
   }
   ```
