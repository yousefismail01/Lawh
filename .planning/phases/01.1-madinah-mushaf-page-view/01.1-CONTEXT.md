# Phase 01.1: Madinah Mushaf Page View - Context

**Gathered:** 2026-03-04
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the ayah-card list with a true 604-page Madinah Mushaf layout as the primary Quran reading view. The mushaf becomes the Home tab experience. Users see the Quran exactly as the printed Madinah Mushaf — 15 lines per page, ornamental frames, decorative surah headers, and proper Arabic justification. The existing surah detail (ayah-card) view is kept as an alternate reading mode.

</domain>

<decisions>
## Implementation Decisions

### Page layout
- True 15-line fixed grid per page matching the printed Madinah Mushaf
- Arabic text justified using kashida (tatweel) stretching to fill lines edge-to-edge
- Ornamental Islamic border/frame around each page
- Cream/parchment background as default, with dark mode variant for nighttime reading
- Inline ornamental circle markers with ayah numbers (matching printed mushaf style)
- Full page header: surah name on one side, juz number on the other, page number centered

### Page navigation
- Both horizontal swipe (default, RTL book-like) and vertical scroll supported
- User setting to choose between horizontal and vertical navigation modes
- Horizontal swipe is the default mode
- Jump-to-page: slider/scrubber bar at bottom + tap to open modal with page number input and surah list
- Auto-resume: save last-read page locally, jump straight to it on open

### Surah transitions
- Ornamental surah headers matching the printed Madinah Mushaf exactly (rectangular banner with surah name)
- Bismillah rendered per printed mushaf conventions: centered between header and first ayah, excluded for Al-Fatiha (part of ayah 1) and At-Tawbah (no Bismillah)
- Pages 1 and 2 (Al-Fatiha and start of Al-Baqarah) get special centered layout with extra ornamentation matching the printed mushaf

### Ayah interaction
- Long press to select an ayah and show action options
- Single tap does nothing — keeps reading experience clean
- Actions available on long press: bookmark and show English translation (bottom sheet)

### App integration
- Mushaf view replaces the Home tab as the primary experience
- Surah list accessible via an icon in the mushaf top bar, opens as modal/bottom sheet
- Selecting a surah from the list jumps to its starting page in the mushaf
- Existing surah/[id] ayah-card view kept as an alternate reading mode

### Claude's Discretion
- Page flip animation details (if any) for horizontal swipe
- Exact ornamental frame SVG/image assets and styling
- Dark mode color palette for mushaf background and text
- Slider/scrubber bar visual design
- Bottom sheet design for bookmark + translation actions
- How to source or generate ornamental surah header assets
- Prefetching strategy for adjacent pages

</decisions>

<specifics>
## Specific Ideas

- The mushaf should look and feel like reading the real printed Madinah Mushaf — this is the guiding principle for all visual decisions
- Pages 1-2 must have the distinctive centered layout that the real Madinah Mushaf uses for Al-Fatiha and start of Al-Baqarah
- The reading experience should be uninterrupted — interactions (bookmark, translation) only appear on deliberate long press

</specifics>

## Existing Code Insights

### Reusable Assets
- `AyahText` component: foundational Arabic text primitive with KFGQPCHafs font, RTL writingDirection, lineHeight=2x — can be adapted for mushaf line rendering
- `SurahHeader` component: has Arabic name rendering and Bismillah logic (excludes surah 1 and 9) — logic reusable even if visual style changes
- KFGQPCHafs font: already loaded, Uthmanic Unicode font supporting proper Arabic rendering
- AmiriQuran font: loaded as fallback

### Established Patterns
- Per-component `writingDirection: 'rtl'` (never global I18nManager.forceRTL)
- `lineHeight = fontSize * 2` for tashkeel rendering
- `quranService` as single data access point — needs new `getAyahsByPage(page, riwayah)` method
- Drizzle ORM on expo-sqlite with `page` column already in schema and populated (1-604)
- `useSettingsStore` (Zustand) for user preferences — add navigation mode and last-read page here
- Expo Router with typed routes — new route needed for mushaf view

### Integration Points
- Home tab (`app/(tabs)/index.tsx`): currently surah list — will become mushaf view
- `app/surah/[id].tsx`: kept as alternate view, accessed from surah list modal
- Tab navigator in `app/(tabs)/_layout.tsx`: Home tab icon/label may change
- Settings store: needs navigation mode preference and last-read page persistence
- Supabase + SQLite `ayahs` table: `page` field (1-604) fully seeded for Hafs riwayah
