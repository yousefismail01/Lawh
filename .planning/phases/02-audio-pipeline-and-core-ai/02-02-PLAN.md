  <action>
    1. Create arabic_normalize.py: Exact Python port of lawh-mobile/lib/arabic/normalize.ts. Same regex patterns, same substitutions, same order. Use unicodedata.normalize('NFC'). Verify cross-language consistency.

    2. Create word_differ.py: `compute_word_diff(transcription_words: list[str], expected_ayahs: list[dict], riwayah: str) -> tuple[list[WordResult], dict]`
       - Split expected text into words preserving tashkeel
       - Align transcription words against expected words using difflib.SequenceMatcher on normalized forms
       - For each alignment pair, classify using this logic:
         a. Both present + normalized match + original match -> CORRECT
         b. Both present + normalized match + original mismatch -> TASHKEEL_ERROR
         c. Both present + normalized mismatch -> WRONG
         d. Expected word with no match -> SKIPPED
         e. Transcription word with no expected match -> ADDED
       - Return list[WordResult] and word_count_breakdown dict

    3. Create scoring.py: `calculate_accuracy_score(word_results: list[WordResult]) -> float`
       - Phase 2 formula: word accuracy only (tajweed weight is 0 until Phase 3)
       - Score = (correct_count + 0.5 * tashkeel_error_count) / total_expected_words
       - Clamp to [0.0, 1.0]

    4. Create passage_detector.py:
       - On module load, read quran-text-normalized.json (pre-computed normalized corpus)
       - Build inverted index: normalized_word -> list of (surah_id, ayah_number, word_position)
       - `detect_passage(transcription: str, riwayah: str) -> PassageMatch`
       - PassageMatch dataclass: surah_id, start_ayah, end_ayah, confidence, matched_ayahs (list of dicts with text)
       - Algorithm: (a) Normalize transcription, split into words. (b) Look up first 3 words in inverted index to get candidate ayah locations. (c) For top candidates, compute edit distance ratio of full transcription against consecutive ayah range. (d) Return best match above 0.6 threshold. (e) If no match above threshold, raise HTTPException(422, "Could not identify recited passage").

    5. Create data/quran-text-normalized.json: Script or inline code to generate this from lawh-mobile/assets/quran-text.json at server startup. Structure: list of {surah_id, ayah_number, text (with tashkeel), normalized_text}. For now, create a stub loader that reads from the mobile assets path (the actual generation happens at startup or as a build step). Include a fallback that loads from a bundled JSON if present.

    6. Write all tests per behavior list. Use pytest fixtures with known Quran text snippets (Al-Fatiha ayahs) for deterministic testing. Mock the Quran corpus in passage detector tests.
