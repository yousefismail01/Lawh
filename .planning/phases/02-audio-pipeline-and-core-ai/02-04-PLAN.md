    1. Create services/inferenceService.ts:
       - `submitRecitation(filePath: string, riwayah: Riwayah): Promise<RecitationResult>`

       - Get access_token from useAuthStore.getState().session
       - Build FormData with audio file (uri, type: 'audio/wav', name: 'recitation.wav') and riwayah
       - POST to ${API_BASE}/inference/recitation with Authorization: Bearer header

       - TanStack Query mutation hook wrapping submitRecitation
       - Manages the upload -> analyze -> results lifecycle

       - On error: store.setError(error.message, filePath)

    3. Create components/mushaf/RecordButton.tsx:
       - Replace MicPlaceholderButton with functional recording button
       - Uses useRecording hook for start/stop
       - Visual states:
         a. Idle: Mic icon (Ionicons mic-outline) with accent color, same style as current placeholder
         b. Recording: Pulsing red circle with stop icon (Ionicons stop), animated with react-native-reanimated (scale pulse 1.0 -> 1.15 -> 1.0 loop)
       - Tap handler: if idle, start recording; if recording, stop recording
       - Request microphone permission on first tap (expo-av Audio.requestPermissionsAsync)
       - On permission denied: show toast "Microphone permission required"

    4. Create components/mushaf/WaveformVisualizer.tsx:
       - Renders above the MushafFooter when status === 'recording'
       - Shows: animated waveform bars + elapsed time counter (MM:SS format)
       - Waveform: Row of ~30 vertical bars, each bar height driven by audioLevels from recordingStore

       - Each bar is an Animated.View with height interpolated from audio level (min 4px, max 40px)
       - Bars shift left as new levels push in (sliding window effect)
       - Elapsed time: reads duration from recordingStore, formatted as MM:SS
       - Dark semi-transparent background (rgba(0,0,0,0.7)) with rounded corners
       - Slides up with reanimated entering/exiting animation
