- Handle the footnote objects: `t` array items that are objects with `f` key are footnote markers — skip them for plain text display

- Use `expo-sqlite` to open this DB as a read-only second database (NOT the same connection as the quran DB)
- DB schema: `surah_list` table (surah_number, audio_url, duration) and `segments` table (surah_number, ayah_number, duration_sec, timestamp_from, timestamp_to, segments)
- Export `openAudioDb(): Promise<SQLiteDatabase>` — opens the DB, caches the connection
- Export `getAyahAudioSegment(surahId: number, ayahNumber: number): Promise<{ audioUrl: string, startMs: number, endMs: number } | null>` — queries segments for timestamp_from/to, queries surah_list for audio_url
- Audio URLs follow pattern: `https://download.quranicaudio.com/qdc/mishari_al_afasy/streaming/mp3/{surah}.mp3`
